# AI Agents: A Literature Review

## From Classical Multi-Agent Systems to LLM-Based Autonomous Agents

*March 2025*

---

## 1. Introduction

The concept of an "agent" — an autonomous entity that perceives its environment, reasons about goals, and takes actions — has deep roots in AI, stretching back to the 1950s. What has changed dramatically since 2022 is that large language models have become the reasoning engine at the center of agent architectures, creating a new design space that borrows from classical agent theory while introducing fundamentally different capabilities and failure modes.

This report traces the arc from classical agent foundations through the modern LLM-based agent explosion, examining what the field has learned about architecture, memory, planning, tool use, multi-agent coordination, and evaluation. It draws on survey papers, foundational research, and practical experience from deployed systems.

---

## 2. Classical Agent Foundations

### 2.1 The BDI Architecture

The Belief-Desire-Intention (BDI) model, developed by Rao and Georgeff (1995), remains the most influential framework for reasoning about agents. Based on Michael Bratman's philosophical theory of human practical reasoning, BDI models an agent through three mental attitudes:

- **Beliefs**: The agent's informational state about the world — notably using "belief" rather than "knowledge" to acknowledge that what an agent holds may be false or change.
- **Desires**: Motivational states representing objectives the agent would like to achieve.
- **Intentions**: Committed plans of action that constrain further deliberation and persist over time.

The critical insight of BDI is the separation of *plan selection* from *plan execution*. An agent must balance deliberation time (choosing what to do) against execution time (doing it). BDI provides formal mechanisms for this through commitment strategies — a "blindly committed" agent never revises intentions, while a "single-minded" agent drops commitments only when they become unachievable.

BDI was successfully deployed in real systems, most famously an air traffic management system (OASIS) and a fault diagnosis system for the Space Shuttle. However, it has notable limitations: no learning mechanism, no way to adapt from past behavior, and plans must be pre-authored by designers rather than generated by the agent itself. These limitations would prove relevant decades later when LLM-based agents attempted to solve the same problems with generative approaches.

### 2.2 Reactive vs. Deliberative Agents

Rodney Brooks' subsumption architecture (1986) represented a radical alternative to deliberative planning. Instead of maintaining internal symbolic models and planning against them, reactive agents couple sensory information directly to action selection through layered behaviors. Lower layers handle primitive responses (obstacle avoidance), while higher layers can subsume lower ones for more complex behavior. Brooks' provocative claim: "the world is its own best model."

Reactive agents excelled in unstructured, dynamic environments — robotics, real-time tasks — precisely because they avoided the computational overhead of maintaining and reasoning about world models. But they could not do anything requiring memory, language understanding, or long-horizon planning.

The resolution came through **hybrid architectures** combining a reactive layer for real-time responses with a deliberative layer for goal-directed planning. This layered pattern — fast reactive responses backed by slower strategic reasoning — reappears in modern LLM agent designs, where tool calls and API interactions happen reactively within a higher-level planning loop.

### 2.3 Cognitive Architectures: SOAR and ACT-R

SOAR (Laird, Newell, Rosenbloom) and ACT-R (Anderson) are cognitive architectures that attempted to build general intelligent agents grounded in theories of human cognition. Though they pursued different primary goals — SOAR targeting general AI, ACT-R targeting psychological modeling — they converged on a "common model of cognition" comprising:

- **Perception** for transducing external input
- **Working memory** for task-relevant state
- **Procedural memory** for "how-to" knowledge (production rules)
- **Declarative memory** for facts and experiences
- **Motor/action** interfaces for environment interaction

SOAR was deployed in game AI (StarCraft, Quake II), military simulations (DARPA's Synthetic Theater of War), and virtual human systems at USC's Institute for Creative Technologies. ACT-R saw applications in human performance modeling for the Air Force Research Laboratory.

The influence on modern LLM agents is structural. The memory taxonomy (working/procedural/declarative/episodic) directly maps to how contemporary agent frameworks organize their memory systems. SOAR's spreading activation mechanism for retrieving relevant memories from context finds echoes in RAG-based retrieval approaches. The key difference: classical cognitive architectures relied on hand-authored knowledge, while LLM agents derive their "knowledge" from pretrained weights and retrieved context.

## 3. The Emergence of LLM-Based Agents

### 3.1 ReAct: The Foundational Pattern

The ReAct paper (Yao et al., 2022; published at ICLR 2023) established the foundational pattern for LLM agents. Its core insight was that reasoning and acting, previously studied as separate capabilities, could be synergized by interleaving them in a Thought-Action-Observation loop:

1. **Thought**: The model analyzes the current situation, generating a reasoning trace.
2. **Action**: Based on reasoning, the model takes a domain-specific action (e.g., search, navigate).
3. **Observation**: The model receives feedback from the environment.

This cycle repeats until task completion. The synergy is bidirectional: reasoning traces help the model create, maintain, and adjust action plans (reason to act), while actions bring in external information to ground the reasoning (act to reason).

On HotPotQA and FEVER, ReAct overcame hallucination problems inherent to chain-of-thought reasoning by grounding in external data via a Wikipedia API. On ALFWorld and WebShop (interactive decision-making), ReAct outperformed imitation and reinforcement learning methods by 34% and 10% absolute success rate, using only one or two in-context examples.

ReAct's significance extends beyond its benchmark results. It established the agent loop pattern — observe, think, act, observe — that virtually every subsequent LLM agent framework adopted. It also demonstrated that a frozen language model with few-shot prompting could function as a reasonable agent without any fine-tuning.

### 3.2 Toolformer: Self-Taught Tool Use

Toolformer (Schick et al., 2023; NeurIPS 2023) from Meta addressed a paradox: LLMs excel at complex reasoning but struggle with basic tasks like arithmetic or factual lookup where simple tools would suffice. Toolformer trained models to decide *when* to call tools, *which* tools to call, and *how to incorporate results* — all in a self-supervised manner.

The training approach was clever: the model generates candidate API calls for training texts, executes them, and retains only calls that reduce the model's loss on subsequent tokens. The resulting model learned to invoke calculators (for 97.9% of math examples), Q&A systems (98.1% of factual lookup examples), search engines, translation systems, and calendars — all without task-specific prompting.

Toolformer's contribution was demonstrating that tool use could be *learned* rather than merely *prompted*. This influenced the trajectory toward function calling as a native model capability, which OpenAI, Anthropic, and Google would later build into their APIs as first-class features.

### 3.3 The Autonomous Agent Wave: AutoGPT and BabyAGI

In early 2023, AutoGPT (Toran Bruce Richards, March 2023) and BabyAGI (Yohei Nakajima) ignited massive public interest in autonomous AI agents. AutoGPT accumulated over 100,000 GitHub stars within months. BabyAGI went viral as a 105-line Python script demonstrating autonomous task management.

**What the concepts demonstrated:**
- LLMs could be framed as general problem solvers with planning, subgoal decomposition, and reflection.
- Chaining LLM calls with tool access created systems that felt genuinely autonomous.
- The architecture was accessible — BabyAGI was just 105 lines of code.

**What failed in practice:**
- **Compounding errors**: AutoGPT relied on its own feedback, and errors cascaded without human correction. It hallucinated not just information but *capabilities it did not have*, then attempted to act on them.
- **Infinite loops**: Without adequate stopping conditions, agents got stuck repeating the same failed subtasks.
- **No self-awareness of limitations**: BabyAGI's default goal "Solve World Hunger" exemplified the problem — agents would propose tasks like "collaborate with world governments" that a program obviously cannot execute.
- **Context window exhaustion**: Andrej Karpathy identified the finite context window as the fundamental bottleneck, causing agents to forget prior work and go "off the rails."
- **Prohibitive costs**: The recursive API calling pattern meant significant OpenAI API costs for minimal useful output.
- **Over-engineered memory**: Vector database integration turned out to be overkill; typical agent runs did not generate enough data to justify it. By late 2023, AutoGPT removed external vector DB support.

The 2023 wave was ultimately a proof-of-concept moment. It demonstrated both the promise and the severe limitations of giving LLMs unconstrained autonomy, and set the stage for more disciplined, human-in-the-loop architectures.

## 4. Architectural Patterns in Modern Agents

### 4.1 The Standard Architecture

Lilian Weng's influential blog post "LLM Powered Autonomous Agents" (June 2023) crystallized the emerging consensus on agent architecture around three components:

1. **Planning**: Task decomposition (Chain of Thought, Tree of Thoughts), self-reflection, and strategy adjustment.
2. **Memory**: Short-term (context window / working memory) and long-term (external vector stores for retrieval).
3. **Tool Use**: External APIs, code execution, search engines, databases — anything beyond the model's own weights.

The major survey by Wang et al. (2023, arXiv:2308.11432) — the first published survey in the field — formalized this into four modules: a **profile module** (agent identity/role), a **memory module**, a **planning module**, and an **action module**. This taxonomy has been widely adopted.

### 4.2 Planning Approaches

**Chain of Thought (CoT)** prompting (Wei et al., 2022) established that instructing models to "think step by step" improves reasoning on complex tasks. But CoT is linear — it commits to one reasoning path without exploring alternatives.

**Tree of Thoughts (ToT)** (Yao et al., NeurIPS 2023) generalized CoT by allowing exploration over multiple reasoning branches with self-evaluation and backtracking. Drawing on classical AI search (breadth-first, depth-first) and Kahneman's dual-process theory (System 2 deliberation), ToT dramatically improved performance on tasks requiring exploration: Game of 24 went from 4% (CoT) to 74% (ToT) success rate with GPT-4. The tradeoff is computational cost — ToT requires many more LLM calls.

**LLM+P** approaches outsource planning to classical planners using PDDL (Planning Domain Definition Language), leveraging decades of planning research while using LLMs for natural language understanding and domain translation.

In practice, most production agent systems use relatively simple planning: sequential task decomposition with optional retry/reflection loops. The more sophisticated approaches (ToT, full MCTS-style search) remain largely in research due to cost and latency constraints.

### 4.3 Reflection and Self-Correction

**Reflexion** (Shinn et al., NeurIPS 2023) introduced "verbal reinforcement learning" — instead of updating model weights, agents reflect on failures in natural language and store these reflections in episodic memory for future trials. The framework has three components: an Actor (generates actions), an Evaluator (scores outputs), and a Self-Reflection model (generates verbal feedback).

Results were striking: 91% pass@1 on HumanEval coding benchmark, surpassing GPT-4's 80%. The key insight is that self-reflection serves as a "semantic gradient signal" — providing concrete direction for improvement without requiring parameter updates.

This pattern — try, evaluate, reflect, retry — has become standard in agentic systems. It mirrors how humans learn complex tasks through iterative refinement and is one of the clearest advantages LLM agents have over classical agents, which lacked mechanisms for learning from experience.

### 4.4 Memory Architecture

Memory in LLM agents maps roughly to the cognitive architecture taxonomy:

- **Short-term / Working memory**: The context window itself, plus any in-context examples or recent history.
- **Semantic memory**: Persistent factual knowledge, often stored in vector databases and retrieved via RAG.
- **Episodic memory**: Records of past interactions and experiences, enabling learning from prior successes and failures.
- **Procedural memory**: Stored procedures, tools, and action patterns.

Recent systems have grown sophisticated. MemoryBank implements dynamic memory strength based on the Ebbinghaus Forgetting Curve. A-Mem (Agentic Memory) introduces iterative retrieval-reasoning cycles to overcome RAG's limitations with ambiguous queries. MemGPT uses cache-like architectures to prioritize recent information. MIRIX proposed a six-component architecture combining all memory types with multimodal data handling.

Key findings from memory research (survey by ACM TOIS, 2024):
- Memory-augmented approaches reduce token usage by over 90% while maintaining competitive accuracy.
- Smaller models benefit most from RAG; larger instruction-tuned models benefit more from episodic and semantic memory.
- Agents exhibit a problematic "experience following property" — they learn from past successes but also propagate past errors, leading to self-degradation over time.
- Intelligent forgetting remains an open problem: determining which memories can be safely discarded requires understanding their future relevance.

### 4.5 Tool Use and Function Calling

Tool use has evolved from a research technique to a standardized API feature. OpenAI introduced function calling in mid-2023, followed by Anthropic and Google. The pattern is now standard: the model receives tool schemas (JSON Schema format), decides when to invoke tools, constructs structured arguments, and incorporates results into its reasoning.

Anthropic's Model Context Protocol (MCP), introduced November 2024, represents the push toward standardization — an open protocol for structuring how LLM applications interact with external tools and data sources. MCP is gaining adoption as SaaS providers expose their services to LLM agents through it.

The industry has converged on a common pattern but with provider-specific implementation differences that still require separate code paths — a friction point that MCP aims to resolve.

## 5. Multi-Agent Systems

### 5.1 Frameworks

The multi-agent framework landscape has diversified significantly:

**CrewAI** uses a role-based model inspired by organizational structures, with Crews (dynamic agent collaboration) and Flows (deterministic task orchestration). Fast to set up, good for structured workflows like content production and report generation.

**LangGraph** extends LangChain with graph-based workflow orchestration — nodes represent functions/tools, edges handle conditional transitions. Reached v1.0 in late 2025 and became the default runtime for LangChain agents. Steeper learning curve but maximum control over complex stateful pipelines.

**AutoGen** (Microsoft) focuses on conversational multi-agent architecture with event-driven messaging. Excels at group decision-making, debate scenarios, and mixed-initiative interactions. Used in production at organizations like Novo Nordisk.

**MetaGPT** assigns software development roles (Product Manager, Architect, Engineer) to agents following Standardized Operating Procedures (SOPs). Effective for generating structured artifacts — code, architecture docs, requirement documents — but narrow in scope.

**CAMEL** (Li et al., NeurIPS 2023) introduced a role-playing framework for studying autonomous cooperation between communicative agents. Its contribution was more scientific than practical: generating large conversational datasets to study multi-agent behavior, identifying challenges like "role flipping" where agents swap assigned roles during interaction.

### 5.2 Key Trends

There is convergence toward **graph-based orchestration** across frameworks. Multi-agent systems that encourage extensive agent interaction (debates, iterative planning) incur proportionally higher costs — a CrewAI crew with 5 agents can cost 5x a single agent per task. The cost-capability tradeoff is a real constraint on multi-agent deployment.

## 6. Evaluation and Benchmarks

### 6.1 Major Benchmarks

**SWE-bench** (Princeton, 2023) evaluates whether agents can resolve real GitHub issues by producing patches that pass test suites. SWE-bench Verified (500 human-validated samples) has become the standard for coding agent evaluation. As of early 2026, top systems (Claude Opus 4.5) achieve ~80% on SWE-bench Verified.

**WebArena** (Zhou et al., 2023) tests autonomous web agents on 812 tasks across realistic web environments (e-commerce, forums). Progress has been dramatic: from 14% success rate in 2023 to ~60% by 2025, driven by a convergence on modular Planner-Executor-Memory architectures and specialized training data.

**AgentBench** (Liu et al., 2023) evaluates across eight diverse environments including OS interaction, databases, knowledge graphs, and games. It revealed significant capability gaps between commercial and open-source models.

### 6.2 Evaluation Gaps

Current benchmarks share critical blind spots:
- **Cost is ignored**: Leading agents exhibit 50x cost variations ($0.10 to $5.00 per task) for similar accuracy, yet no major benchmark reports cost metrics.
- **Safety and security**: Prompt injection resistance, organizational policy compliance, and graceful error handling are not systematically evaluated.
- **Enterprise readiness**: A reported 37% performance gap exists between lab benchmark scores and production deployment performance.
- **Long-horizon reliability**: Most benchmarks test individual task completion, not sustained agent performance over extended interactions.

## 7. Recent Developments (2024-2025)

### 7.1 Computer Use

Anthropic's Computer Use (October 2024) allowed Claude to interpret screen content and simulate keyboard/mouse input — essentially using computers the way humans do. Initial success rates on benchmarks like OSWorld were modest (~15%), but by late 2025 with Claude 4 and Sonnet 4.5, rates climbed to the high 80s for standard office tasks.

OpenAI's Operator (January 2025) took a similar approach with the Computer Using Agent (CUA) model, combining GPT-4o vision with reinforcement learning for GUI policies in a secure virtual browser.

Three competing philosophies have emerged: OpenAI's programmable substrate (developer-first), Google's governed enterprise scale (Vertex AI integration), and Anthropic's human-in-the-loop approach. As one analysis put it, "trust, not just capability, remains the bottleneck for AI adoption."

### 7.2 Agentic Coding

The agentic coding space has matured rapidly:

- **Claude Code**: Anthropic's terminal agent. Solves 72% of medium-complexity GitHub issues in under 8 minutes on SWE-bench Verified. Full codebase awareness, multi-file edits, shell access, iterative test-and-fix loops. "If Cursor is about flow, Claude Code is about intelligence."
- **Cursor**: IDE-based (VS Code fork) with Agent mode. Dominates developer workflow for its visual feedback loop and autocomplete. 40,000 enterprise users by 2025. Weaker on large, complex refactors.
- **Devin**: Fully autonomous cloud-based developer. Highest autonomous SWE-bench score at launch (13.9% on original SWE-bench). Opens PRs, auto-fixes CI failures. But expensive and still makes "rookie mistakes" like storing plaintext passwords.
- **SWE-Agent**: Princeton's pioneering open-source framework, instrumental in establishing SWE-bench. Since surpassed by commercial tools but influential in proving the concept.

The consensus from practitioners: "The agents handle the typing. The engineer handles the thinking." Multi-tool strategies are common — IDE tools for daily flow, terminal agents for complex tasks, enterprise tools for compliance.

### 7.3 Agent Safety and Alignment

The safety landscape has grown urgent. 80% of organizations report encountering risky behaviors from AI agents, including unauthorized data exposure and improper system access. Key concerns:

- **Digital insider threat**: Agents operate within systems with privilege, like human employees, and can cause harm through misalignment or compromise.
- **Sandboxing is rare**: Only 9 of 30 agents in a 2025 survey documented sandboxing; 9 had no guardrails documented at all.
- **Cascading failures**: Greater autonomy means a single compromise can propagate across business-critical systems.
- **Capability overhang**: Features enabling defensive coordination (planning, memory, tool orchestration) can equally enable offensive operations — autonomous reconnaissance, adaptive exploitation, coordinated social engineering.

OWASP published a taxonomy of 15 threat categories for agentic AI. AWS proposed a four-scope security framework based on autonomy and agency levels. The McKinsey playbook for deploying agentic AI emphasizes Zero Trust principles applied to non-human entities.

A strong argument has been made that "fully autonomous AI agents should not be developed" (arXiv:2502.02649), advocating for structured autonomy levels with human-in-the-loop checkpoints rather than unconstrained agency.

## 8. What Actually Works vs. What Is Hype

### What works:
- **Constrained, well-defined tasks**: Agents excel when the action space is bounded, goals are clear, and evaluation criteria are unambiguous (coding, web navigation, database queries).
- **Human-in-the-loop architectures**: The most successful deployed systems keep humans in the decision loop, using agents for execution rather than autonomous decision-making.
- **Tool use via function calling**: Standardized, well-documented tool interfaces dramatically improve agent reliability.
- **Reflection with bounded retries**: Reflexion-style try-evaluate-retry loops work when there is a clear signal for success/failure (e.g., tests passing).
- **Specialized agents over general ones**: Agents trained or prompted for specific domains consistently outperform general-purpose autonomous agents.

### What remains hype or immature:
- **Fully autonomous agents**: The AutoGPT lesson still holds — unconstrained autonomy leads to loops, hallucinations, and wasted compute.
- **Complex multi-agent debate/coordination**: In practice, the overhead and cost of multi-agent interaction often exceeds the benefit for most tasks.
- **Long-horizon autonomous planning**: Agents still struggle to maintain coherent plans over dozens of steps without human guidance.
- **Self-improving agents**: While Reflexion shows promise, true self-improvement that does not degrade over time remains elusive. The "experience following property" means agents can learn bad habits as readily as good ones.

## 9. Open Problems and Limitations

1. **Error propagation in chains**: Unlike humans who naturally filter information, LLM agents amplify errors through reasoning chains rather than correcting them.
2. **Memory management**: Intelligent forgetting, memory consolidation, and preventing self-degradation from accumulated experience remain unsolved.
3. **Autonomy-control tradeoff**: Organizations lose fine-grained control in autonomous systems while struggling to reason about overall system behavior.
4. **Cost at scale**: Sophisticated agent architectures (multi-agent, ToT-style search, extensive reflection) are expensive to run. Cost-performance optimization is underdeveloped.
5. **Security**: The attack surface of agentic systems is fundamentally larger than that of single-turn LLM inference, and defensive tooling is immature.
6. **Evaluation**: Current benchmarks do not capture cost, safety, long-horizon reliability, or enterprise deployment realities.
7. **Knowledge currency**: Agents reason from pretrained weights and retrieved context, both of which can be stale. Keeping agent knowledge current without retraining remains a challenge.

---

## 10. Where the Field Is Heading

The trajectory is toward **structured autonomy**: agents that are powerful but constrained, with clear boundaries, human oversight checkpoints, and well-defined tool interfaces.

MCP is becoming the standard for tool and data access. Graph-based orchestration is converging across frameworks. Inference-time scaling (spending more compute at generation time) is expected to be a major lever for 2026-2027.

The most productive framing is not "autonomous AI agents" but "AI-augmented workflows" — systems where agents handle execution within bounds set by human operators. The 2023 dream of fully autonomous agents that solve arbitrary problems has given way to a more sober understanding: agents are powerful tools that amplify human capability, not replacements for human judgment.

The classical agent foundations — BDI's separation of deliberation from execution, cognitive architectures' memory taxonomies, hybrid reactive-deliberative designs — turn out to be remarkably relevant. The field has, in some sense, spent 2023-2025 rediscovering through empirical experience what agent researchers theorized in the 1990s, now with LLMs as the reasoning substrate rather than hand-authored rules.
