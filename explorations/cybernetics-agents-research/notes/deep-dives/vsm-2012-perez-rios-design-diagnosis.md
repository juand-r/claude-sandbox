# José Pérez Ríos — VSM in Practice: Design, Diagnosis, and Pathologies

## Author Profile

Full Professor, Universidad de Valladolid. Developed practical methodology and software tools for VSM application. Key figure in bridging Beer's theory and practical implementation.

## Key Works

### Book: Design and Diagnosis for Sustainable Organizations: The Viable System Method (Springer, 2012)
- Practical guide combining systems theory with cybernetics
- Provides "guidelines for practical application" — filling the gap between Beer's theory and real-world use
- "An ideal primer to enable both understanding and adequate application of the VSM"

### Paper: Schwaninger & Pérez Ríos (2010) — "Models of Organizational Cybernetics for Diagnosis and Design"
- Kybernetes, Vol. 39, No. 9-10, pp. 1529-1550

### Paper: Pérez Ríos (2024/2025) — VSM and AI Governance
- Uses VSM + TOP (Taxonomy of Organizational Pathologies) to address AI complexity
- Concepts: "variety engineering," "recursive structure design," "governance"

## Four-Stage Methodology

1. **Identity:** Clarify the identity, purpose, and boundaries of the organisation
2. **Vertical structure:** Create the vertical structure to cope with environmental complexity (recursion levels)
3. **VSM diagnosis:** Design or diagnose each organisation following VSM indications
4. **Coherence check:** Verify coherence across all recursion levels — identity and purposes must be consistent at every level

## Recursion Levels–Key Factors Matrix

Developed by Pérez Ríos (2008, 2010, 2012):
- Rows = recursion levels
- Columns = key factors (selected case-by-case)
- Makes the entire recursive structure visible at a glance
- Facilitates coherence between actions at all levels

This is an important practical contribution: it provides a *visual tool* for managing multi-level recursion, something Beer's theory describes but does not provide a concrete representation for.

## Taxonomy of Organizational Pathologies (TOP)

A systematic categorisation of what goes wrong in organisations when VSM principles are violated. Integrated into the diagnostic framework.

### Categories of Pathologies (from Beer, extended by Pérez Ríos):

**System 1 pathologies:**
- Operational elements lack viable-system structure
- Insufficient autonomy
- Overlapping environments without coordination

**System 2 pathologies:**
- Missing coordination mechanisms → uncontrolled oscillations
- System 2 used for command rather than regulation

**System 3 pathologies:**
- Over-control (excessive intervention in System 1 autonomy)
- Under-control (failure to maintain cohesion)
- Confusion with System 5 functions

**System 3* pathologies:**
- Absent audit → System 3 depends on biased self-reports
- Excessive audit → destroys autonomy

**System 4 pathologies:**
- Absent or weak → organisation blind to environmental change
- Root of most major business collapses (Beer's empirical claim)

**System 5 pathologies:**
- Somnolence (lulled by well-functioning 3-4 homeostat)
- Identity crisis
- Intrusion into System 3

**Cross-system pathologies:**
- 3-4 homeostat failure (the "yo-yo pathology")
- Recursion incoherence (purposes misaligned across levels)

## VSMod® Software Tool

- Developed by Systems Thinking Group at Universidad de Valladolid
- Allows graphical, interactive creation of VSM models
- Handles cases with multiple units and recursion levels
- Repository for System Dynamics models
- Bridges VSM (structure) with SD (dynamic behaviour)

## System Dynamics + VSM Integration

Pérez Ríos advocates bridging System Dynamics (SD) with VSM:
- VSM defines the organisational structure
- SD models the dynamic behaviour within that structure
- Together they provide both structural diagnosis and dynamic simulation

## Relevance to Agent Architectures

1. The four-stage methodology maps directly to agent system design
2. The Recursion Levels–Key Factors Matrix is a practical tool for designing multi-level agent hierarchies
3. TOP provides a checklist of failure modes to design against
4. The VSM+SD bridge suggests agent architectures should combine structural design (VSM-like) with behavioural simulation (SD-like)
5. VSMod as a software tool demonstrates that VSM-based design can be computationally supported
