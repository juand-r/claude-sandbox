# Applying Radical Constructivism to Machine Learning: A Pilot Study in Assistive Robotics

**Authors:** Markus Nowak, Claudio Castellini, Carlo Massironi
**Year:** 2018
**Journal:** Constructivist Foundations, Vol. 13, No. 2, pp. 250-262
**URL:** https://constructivist.info/13/2/250.nowak
**Type:** Target article

## Core Contribution

Matches machine learning (ML) and interactive machine learning (iML) with radical constructivism (RC) to build a framework for interactive ML applied to assistive robotics (upper-limb prosthetics).

## Problem

- Myocontrol (prosthetic control via muscle signals) is "still unsolved, with rejection rates of up to 75%"
- Traditional ML approaches treat the user as a fixed signal source rather than an adaptive agent
- This is fundamentally a constructivist problem: the system and user must co-construct a shared understanding

## Key Arguments

- Ernst von Glasersfeld's radical constructivism serves as "the leading principle pushing for the usage of RC-framed iML"
- Emphasizes **co-adaptation** between human users and prosthetic systems — mutual learning rather than one-directional machine learning
- The constructivist perspective enables "principled design of the system itself, as well as of the training/calibration/co-adaptation procedure"
- Pilot study compared RC-framed interactive ML system against conventional ML approaches

## Relevance to Project

This paper demonstrates that constructivist epistemology has practical engineering consequences. The co-adaptation framing (both agent and environment/user adapt simultaneously) is directly relevant to agent design. It shows RC isn't just philosophical hand-waving — it changes how you build the system.

The rejection rate problem (75%!) is a case study in what happens when you ignore the constructivist insight that knowledge is constructed through interaction, not extracted from a pre-given world.
