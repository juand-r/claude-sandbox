## 1. The Core Claim: Organisms Control Perceptions, Not Outputs

### The Central Thesis

PCT's foundational claim inverts the standard picture of behavior:

> "The thesis of PCT is that animals and people do not control their behavior; rather, they vary their behavior as their means for controlling their perceptions, with or without external disturbances." (Powers, 1973)

The controlled variable is not the output of the system (behavioral actions), but its **input** -- a sensed and transformed function of some state of the environment. Because these sensed inputs may appear as consciously perceived aspects of the environment, Powers labelled the controlled variable "perception."

### Why This Matters: The Formal Argument

1. **Behavior is inherently variable.** The same goal (e.g., staying in a lane while driving) is achieved through constantly varying actions (steering adjustments). No fixed output pattern exists.

2. **What is invariant is the perception.** Despite disturbances (wind gusts, road curvature), the *perceived position relative to lane center* remains stable. The perception is what is maintained -- it is the controlled variable.

3. **The output cannot be the controlled variable** because the same perception can be maintained by entirely different output patterns depending on the disturbances present. Outputs vary; perceptions are held constant. What is held constant is what is controlled.

4. **Circular causation, not linear.** Actions change the environment, which changes perception, which changes actions. There is no starting point. This is a closed causal loop, not a linear stimulus-to-response chain.

### Key Predictions Generated by This Claim

- **Hypothesis 1:** If perceptual input is controlled at a fixed value through dynamic action, there will be no reliable statistical relationship between input and output (because output varies to compensate for disturbance).
- **Hypothesis 2:** Which perceptual variable is being controlled can be discovered by disturbing the perceptual input in various ways (the Test for the Controlled Variable).
- **Hypothesis 3:** Computer models based on PCT can match the behavior of individual humans in real-world tasks.

## 2. The Negative Feedback Control Loop

### Structure

The basic unit of PCT is a negative feedback control loop with the following components:

```
                    ENVIRONMENT
                   /           \
                  /             \
    [Output      /               \   [Disturbance d]
     affects    /                 \   acts on
     environment]                  \  environment
                                    |
              q.o                 q.i (environmental
           (output                    input quantity)
            quantity)                     |
                |                         |
                |                    [Input Function I]
                |                    transforms q.i into
                |                    perceptual signal p
                |                         |
                |                         v
         [Output Function O]        +-----------+
         generates output      r -->| Comparator|
         from error signal          |   C       |
                ^                   |  e = r - p|
                |                   +-----------+
                |                         |
                +--- e (error signal) ----+

    r = reference signal (desired state)
    p = perceptual signal (current perceived state)
    e = error signal = r - p
    q.o = output quantity (action on environment)
    q.i = input quantity (state of environment sensed)
    d = disturbance (external perturbation)
```

### Component Functions

| Component | Function | Mathematical Form |
|-----------|----------|-------------------|
| **Input Function (I)** | Transforms environmental variable q.i into perceptual signal p | p = I(q.i) |
| **Comparator (C)** | Computes discrepancy between reference and perception | e = r - p |
| **Output Function (O)** | Converts error signal into output (often via integration) | o = O(integral of e * dt) |
| **Feedback Function (environment)** | The physics by which output affects the input variable | q.i = f(q.o, d) |
| **Reference Signal (r)** | Specifies the desired perceptual state | Set by higher-level system or intrinsic |
| **Disturbance (d)** | External influences on q.i independent of the system's output | Independent variable |

### Key Properties

- **Two independent variables:** the reference signal r and the disturbance d.
- **When the loop has high gain,** the perception p tracks the reference r closely, regardless of disturbances. This is the defining property of control.
- **The output is NOT the controlled variable.** The output varies continuously and unpredictably (from an observer's perspective) because it must compensate for both the reference signal and unknown disturbances.
- **Neural implementation:** Powers (1973) proposed that the comparator's subtraction function is implemented by convergence of excitatory and inhibitory neural connections at the pre-synaptic membrane. Addition, multiplication, and integration are implemented by other known neural structural arrangements.

### The Thermostat Analogy (and Its Limits)

A thermostat illustrates the basic loop:
- Perception = measured temperature
- Reference = dial setting (desired temperature)
- Error = desired minus actual temperature
- Output = signal to heater (on/off or proportional)
- Disturbance = outdoor temperature changes, open windows

But living control systems differ from thermostats in a critical way: **the reference signal is generated internally**, usually as a function of error output from a higher-level system. In a thermostat, the setpoint is externally specified. In an organism, reference signals are generated within the hierarchy.

## 3. The Hierarchy of Perceptual Control (HPCT)

### The 11 Levels

Powers proposed a hierarchy of perceptual control levels, where each level constructs its perceptions from combinations of signals at the level below. The original 9 levels (1973) were later expanded to 11. **Powers repeatedly cautioned that these specific levels are hypotheses, not established fact.** The principle of hierarchical organization matters more than the specific number or naming of levels.

| Level | Name | What Is Controlled | Example |
|-------|------|-------------------|---------|
| 1 | **Intensity** | Magnitude of sensory signals; degree of neural firing | Brightness of light, loudness of sound, muscle tension in a finger |
| 2 | **Sensation** | Combinations of intensities that form recognizable sensory qualities | Color (combination of wavelength intensities), texture, warmth, "the sensation of tension in a muscle" |
| 3 | **Configuration** | Spatial patterns of sensations; shapes, objects, orientations | A face, the angle of an arm, a letter of the alphabet, spatial layout of objects |
| 4 | **Transition** | Rate of change of lower-level perceptions over time; derivatives | Motion, acceleration, change of pitch, rate of temperature change |
| 5 | **Event** | Recognizable sequences of transitions with a beginning and end | A door opening, a ball being thrown, steering a car through a curve |
| 6 | **Relationship** | Perceived connections or contingencies between events or configurations | "If I push this, that moves"; spatial relationships like "above" or "next to"; "driving" as a sustained relational activity |
| 7 | **Category** | Classification of lower-level perceptions into types | "This is a chair"; "that behavior is aggressive"; "motor skills" as a class |
| 8 | **Sequence** | Ordered series of lower-level perceptions | Steps in starting a car; a melody; a recipe; a sentence |
| 9 | **Program** | Contingent sequences with decision points (if-then branching) | "Drive to the store, if it's closed, go to the other one"; navigating a conversation |
| 10 | **Principle** | Abstract rules or values that govern programs and choices | "Meet commitments"; "be honest"; "minimize harm"; heuristics for living |
| 11 | **System Concept** | The most abstract self-referential perceptions; one's model of "how things work" or "who I am" | "Be a responsible person"; "I am a scientist"; a worldview; an ideology |

### How the Hierarchy Works

**Bottom-up (perception):** Each level constructs its perceptual signals by combining (weighting, integrating, transforming) signals from the level immediately below. Level 1 takes raw sensory input from receptors. Level 2 combines intensities into sensations. Level 3 combines sensations into configurations. And so on up.

**Top-down (control):** Higher levels control by setting the reference signals for lower-level control systems. A principle-level system ("be honest") sets reference signals for program-level systems (how to respond in a conversation), which set references for sequence-level systems (the words to say in order), and so on down to intensity-level muscle contractions.

**Only the bottom level acts on the environment.** All higher levels act indirectly, by adjusting goals for the levels below them. The higher systems tell lower systems *what to perceive*, not *what to do*. The lower systems figure out the "when, how much, and which way" autonomously.

**Fanning out:** Reference signals from higher levels fan out to many lower-level systems with varying weights. A single principle can simultaneously set references for dozens of programs.

**Consequence:** No central executive is needed. Complex, coordinated behavior emerges from many simple control loops operating simultaneously, each controlling its own perception. Powers' "Crowd" simulation demonstrated that hundreds of interdependent control loops can be "in control" at the same time with widely varying reference signals.

### Development of the Hierarchy

The lower levels were proposed to be consistent with known neurophysiology. The higher levels were inferred from introspective analysis and behavioral observation. Powers acknowledged this was speculative and called for empirical investigation of the specific levels.

## 4. Why PCT Rejects Stimulus-Response and Plan-Execute Models

### Against Behaviorism (Stimulus-Response)

The behaviorist model: S --> [organism] --> R (linear causation from stimulus to response).

**PCT's objections:**

1. **The quantitative problem.** Behaviorists correctly noted that behavior depends on perception (you cannot drive with your eyes closed). But they could not explain how stimuli could affect responses in exactly the quantitative way needed to keep a car in its lane. The mapping from stimulus to response is not fixed -- it changes moment to moment depending on disturbances. No S-R law can capture this because the *same* stimulus produces different responses depending on current disturbances.

2. **The variability problem.** If behavior were caused by stimuli, then the same stimulus should produce the same response. But it doesn't. The same visual input of a road ahead produces wildly different steering outputs depending on wind, road surface, tire pressure, etc. What remains constant is the *perception* (car centered in lane), not the stimulus-response mapping.

3. **Closed loop, not open loop.** The S-R model is an open-loop causal model. It ignores the fact that the organism's output changes the environment, which changes the stimulus. Behavior and perception are locked in a continuous circular causal loop. You cannot break this loop and analyze either direction in isolation.

4. **Rejection of introspection was premature.** Behaviorists rejected introspective reports to maintain objectivity, but in doing so they eliminated the very data (perception, goals, reference states) that are central to understanding behavior.

### Against Cognitivism (Plan-Execute)

The cognitivist model: S --> [cognitive processing / world model / plan] --> R.

**PCT's objections:**

1. **Still linear causation.** Cognitivism inserted cognitive processing between stimulus and response but retained the fundamental assumption of linear causation from environment to behavior. Marken called this an "open-loop causal model of behavioral organization" -- same structural flaw as behaviorism, just with more steps in the middle.

2. **The computation problem.** To plan actions, the brain would need an impossibly detailed model of the world, including precise knowledge of all current disturbances, the physics of the environment, and the exact state of the body. It would need to perform inverse kinematics, predict the future, and compute optimal trajectories -- all in milliseconds. This is implausible given known neural processing speeds.

3. **We plan results, not actions.** When you decide to pick up a cup, you do not plan the exact sequence of muscle contractions. You set a perceptual goal (cup in hand) and let lower-level control systems figure out the motor details. Even well-designed robots don't work by computing actions from a world model -- they work better using feedback control.

4. **Plans cannot handle disturbances.** Any pre-computed plan will be wrong as soon as an unexpected disturbance occurs. Real behavior shows continuous, real-time adaptation to disturbances -- something that requires closed-loop control, not open-loop plan execution.

### PCT as the Alternative

PCT replaces both frameworks with **closed-loop circular causation**:
- Behavior is not caused by stimuli (behaviorism) or by plans (cognitivism).
- Behavior is the **means** by which organisms control their perceptions.
- The organism does not process inputs to produce outputs. It controls inputs by varying outputs.
- Purpose and intention have objective status in PCT -- they are reference signals in control loops -- without requiring introspection or teleological causation.

## 5. The Test for the Controlled Variable (TCV)

### Rationale

If behavior is organized around the control of perceptual variables, then understanding behavior requires identifying *which* perceptual variable is being controlled. This is the central empirical question of PCT. The TCV is the methodology Powers developed to answer it.

### The Logic

A variable that is under control will be **protected from disturbance** by the actions of the control system. If you disturb a controlled variable, the system will act to counter the disturbance and restore the variable to its reference value. If a variable is *not* being controlled, disturbances will have their full effect -- no compensatory action will occur.

### Procedure

1. **Hypothesize** a perceptual variable that the organism might be controlling.
2. **Apply a disturbance** to that variable (change something in the environment that would alter the hypothesized perception if left uncorrected).
3. **Observe** whether the organism acts to counter the disturbance and maintain the variable near a reference value.
4. **If compensatory action occurs** (the variable remains stable despite disturbance), the hypothesis is supported.
5. **If no compensatory action occurs** (the variable changes freely with the disturbance), the hypothesis is refuted.
6. **Iterate** with different hypothesized variables until the true controlled variable is identified.

### Key Methodological Insight

Standard experimental psychology manipulates independent variables (IVs) and looks for changes in dependent variables (DVs). The TCV does the opposite: **it looks for variables that do NOT change when IVs are manipulated.** Standard research studies variability; PCT studies **invariance**. This is more akin to research in the physical sciences.

### Model-Based TCV

A more rigorous version uses computer simulation (Marken, 2005). A PCT model is built with a hypothesized controlled variable, and the simulation's behavior is compared to human data. The hypothesis that produces the best fit identifies the actual controlled variable. This provides quantitative, not just qualitative, evidence.

### Empirical Applications

The TCV has been applied to:
- Computer tracking tasks (robust results; Marken, 2014)
- Hoarding behavior of rats
- Posture control of crickets
- Parenting behavior of chimpanzees
- Shock avoidance behavior of rats
- Object interception behavior of humans and dogs

## 6. Reorganization: PCT's Theory of Learning

### The Problem

How do control systems come into existence? How are their parameters (gains, reference values, connections) tuned? How do organisms learn new behaviors? PCT cannot appeal to reinforcement of specific responses (since the same goal is achieved by different responses each time). It needs a different learning mechanism.

### The Mechanism: Reorganization

**Reorganization** is PCT's equivalent of learning. It operates as follows:

1. **Intrinsic variables** are physiological quantities essential to survival (blood glucose, body temperature, oxygen levels, etc.). These are controlled by an **intrinsic control system** that is part of the inherited structure of the organism.

2. **When intrinsic variables are out of range** (persistent error in the intrinsic control system), reorganization is triggered.

3. **Reorganization modifies the parameters and connectivity of the control hierarchy** in a **random-walk manner**. It may:
   - Change loop gains (sensitivity, efficiency)
   - Change reference values
   - Create new connections between levels
   - Destroy existing connections
   - Create new perceptual signals (by combining lower-level signals in new ways)
   - Create entirely new control systems

4. **When a modification reduces intrinsic error** (improves survival-relevant variables), the rate of reorganization slows. When intrinsic error increases, reorganization speeds up.

5. **The process halts when intrinsic error reaches zero** (or near-zero). The hierarchy is then configured in a way that successfully controls for the organism's survival needs.

### Relationship to Ashby's Ultrastability

Powers explicitly adopted this mechanism from W. Ross Ashby's **homeostat** and the concept of **ultrastability** (Ashby, 1952, *Design for a Brain*).

Ashby's ultrastability: a system has two feedback loops:
- A **fast inner loop** that controls environmental variables via ordinary negative feedback.
- A **slow outer loop** that modifies the parameters of the inner loop when essential variables go out of range.

The outer loop makes random changes to inner-loop parameters until a configuration is found that keeps essential variables within limits. This is ultrastability -- the system is stable not just against environmental disturbances (which the inner loop handles) but against the failure of the inner loop itself (which the outer loop handles by restructuring).

PCT's reorganization is essentially Ashby's ultrastability applied to a hierarchical control system. The key difference: PCT specifies a **hierarchy** of control levels, so reorganization can modify connections and parameters at any level, not just a single-level controller.

### Later Refinement: E. coli Navigation

Powers later refined the reorganization mechanism based on *E. coli*'s chemotaxis strategy (described by Koshland, 1980):
- E. coli moves in a straight line as long as the nutrient gradient is increasing.
- When the gradient decreases (things are getting worse), it tumbles randomly and sets off in a new direction.
- This produces net movement up nutrient gradients without any representation of the gradient or planning.

Similarly, PCT reorganization: there is a **basic continuous rate of random change** in hierarchy parameters. The speed of change is set by the total intrinsic error. When things are going well (low error), change slows. When things are going badly (high error), change speeds up, with random changes in direction. This is a gradient ascent without representation of the gradient.

### Developmental Progression

Reorganization accounts for the observed developmental sequence of learning:
1. **Trial-and-error learning** (infants): rapid, random reorganization; many parameters changing
2. **Associative learning** (young children): slower, more targeted reorganization at specific levels
3. **Systematic learning** (adults): reorganization guided by higher-level systems that can direct attention to specific areas of the hierarchy

### Therapeutic Implications

When control systems are in **conflict** (two systems trying to control the same variable at different reference values), intrinsic error increases and reorganization is triggered. The **Method of Levels (MOL)** therapy, developed by Powers and elaborated by Carey, guides the client's awareness to the higher-level control systems driving the conflict, so that reorganization can resolve it.

## 7. PCT Robotics

### Architecture

A PCT-controlled robot is organized as a hierarchy of negative feedback control loops:

1. **Only the bottom layer** sends outputs to physical actuators (motors, servos).
2. **All higher layers** send their outputs to the reference inputs of the next level down.
3. Each layer controls its own perceptions by setting goals for the layer below.
4. No inverse kinematics, no world model, no pre-computed trajectories.

### How It Differs from Conventional Control

| Aspect | Conventional Robotics | PCT Robotics |
|--------|----------------------|--------------|
| **What is controlled** | Outputs (joint angles, motor commands) | Perceptions (sensed position, sensed force) |
| **World model** | Required (often detailed) | Not required |
| **Inverse kinematics** | Required | Not required |
| **Disturbance handling** | Requires explicit compensation or replanning | Inherent -- the feedback loop counters disturbances automatically |
| **Computational cost** | High (optimization, trajectory planning) | Low (simple subtraction and integration at each loop) |
| **Robustness** | Brittle in novel situations | Robust -- adapts continuously to unpredictable environments |
| **Complexity of design** | High | Low -- few components per loop |

### Key Advantage: No Inverse Problem

In conventional robotics, to move a hand to a target position, the system must solve the **inverse kinematics problem** -- computing joint angles from desired end-effector position. This is computationally expensive and fragile.

In PCT robotics, the system simply:
1. Perceives current hand position (via sensors)
2. Compares it to desired hand position (reference signal)
3. Generates error signal
4. Outputs corrective action that reduces error

The "inverse problem" is solved implicitly and continuously by the feedback loop, without any explicit computation.

### Implemented Systems

1. **Inverted Pendulum Robot** (Young et al., 2020): First direct comparison of PCT controller vs. classical LQR controller for a two-wheeled balancing robot. PCT performance was comparable to LQR and **superior at disturbance rejection**. The PCT controller used a hierarchy of only 5 simple control systems, compared to the more complex LQR formulation.

2. **Archy (Hexapod)** (Kennaway): A simulated six-legged robot capable of standing, walking over uneven terrain, and climbing stairs. All control systems are simple PID controllers arranged hierarchically. Critically, Archy **continues to function when legs are removed** -- the remaining legs compensate automatically because each controls its own perception, not a fixed output pattern.

3. **Robot Arms** (Adam Matic): Robot arm using visual and pressure sensors, modeling extensor and flexor muscle action using PCT. No inverse kinematics.

4. **Autonomous Rover**: Real-world robotic rover using hierarchical negative feedback control. Behavior emerges as side effects of the control hierarchy, not from explicit programming of behavioral routines.

5. **Crowd Simulation** (Powers): Demonstrates emergent collective behavior from independent agents each controlling simple perceptions. Bo Wang's extension added E. coli-style reorganization, demonstrating learning in artificial PCT agents.

### Fundamental Critique of Conventional Approach

PCT robotics researchers argue that the conventional control-of-output approach is not merely less efficient but **fundamentally misconceived**:

> "Rather than just presenting an improved approach to the development of artificial systems, PCT suggests that the conventional control-of-output approach is misconceived, fundamentally flawed, and mired in unnecessary complexity."

## 8. PCT vs. Reinforcement Learning

### Fundamental Structural Differences

| Dimension | PCT | Reinforcement Learning |
|-----------|-----|----------------------|
| **What is optimized** | Error minimization (perception matching reference) | Cumulative reward maximization |
| **What drives behavior** | Discrepancy between perception and reference | Expected future reward |
| **Controlled quantity** | Perceptual input (sensory signal) | Policy (mapping from states to actions) |
| **Learning mechanism** | Reorganization (random modification of hierarchy parameters when intrinsic error is high) | Policy/value function updates via reward signal (TD learning, policy gradient, etc.) |
| **Internal model** | No world model; no simulation before action | Often learns value functions, state transition models, or both |
| **Hierarchy** | Hierarchy of perceptual levels (what is perceived) | Hierarchy of policies and sub-goals (what is done) |
| **Goal specification** | Reference signals (desired perceptions) at each level | Reward function (external or shaped) |
| **Behavior's role** | Means to an end (controlling perception); inherently variable | The thing being optimized; converges to a policy |
| **Conflict** | Between incompatible reference values for the same perception | Exploration-exploitation dilemma |
| **"Reinforcement"** | No specific response is reinforced; trial-and-error changes to control system properties | Specific actions/policies are reinforced by reward |

### The Deepest Difference

In RL, behavior (the policy) **is** the thing being optimized. The goal is to find the best mapping from states to actions.

In PCT, behavior is **not** the thing being controlled or optimized. Behavior is the **side effect** of controlling perceptions. There is no "best behavior" -- there is only whatever behavior currently reduces perceptual error. The same goal will be achieved by completely different behaviors under different disturbance conditions.

This means PCT has no need for the concept of "optimal action." There is no optimal steering input for staying in a lane -- the optimal input changes every millisecond depending on wind, road surface, and tire conditions. What is optimal is the *control* of the perception, which the loop achieves automatically.

### On Reward

PCT does not use reward signals. The closest analogue is **intrinsic error** -- the discrepancy between intrinsic variable states and their reference values. But this drives *reorganization* (structural modification of the hierarchy), not reinforcement of specific behaviors.

RL's reward is typically externally defined and shapes the policy. PCT's intrinsic error is internally generated and shapes the *architecture*, not any specific behavior.

### Connection to Active Inference

The **Free Energy Principle** (Friston) provides a bridge between these frameworks. Active inference shows that agents minimizing free energy (prediction error) reproduce policies that match RL solutions, without invoking reward, value, or utility. This is structurally closer to PCT than to RL, since the agent minimizes a sensory prediction error rather than maximizing a reward. However, active inference assumes generative models (internal world models), which PCT explicitly rejects.

## 9. Key Criticisms and Open Questions

### Legitimate Criticisms

1. **No theory of information.** PCT describes control of perceptions but does not formally specify how information is structured or how it relates to the dynamics of the world (criticism from ecological psychology).

2. **Unconstrained model fitting.** PCT models have enough free parameters (gains, reference values, time constants) that they can be fit to many datasets without strongly constraining the space of possible systems.

3. **The 11 levels are speculative.** Powers himself acknowledged that the specific number and character of levels are hypotheses requiring empirical validation. Little direct neurophysiological evidence exists for the higher levels.

4. **Experimental evidence is concentrated in tracking tasks.** The strongest quantitative evidence for PCT comes from computer tracking studies. Extension to complex real-world behavior is more qualitative.

5. **Relationship to other control-theoretic approaches** (optimal control, model predictive control, active inference) needs more formal clarification.

### Open Research Questions

- Can the TCV be reliably applied to identify controlled variables in complex, naturalistic behavior?
- What is the neural substrate of the comparator, reference signal, and reorganization system?
- How does the hierarchy develop ontogenetically? Is there a principled developmental sequence?
- Can PCT be integrated with information-theoretic approaches?
- How does PCT scale to social behavior, cultural systems, multi-agent interaction?

---

## 10. Summary: What Makes PCT Distinctive

1. **Organisms control inputs, not outputs.** This single claim, if taken seriously, inverts the entire framework of behavioral science.

2. **Behavior is the means, not the end.** Behavior is inherently variable and unpredictable because it is the continuously varying means by which stable perceptions are maintained.

3. **Circular causation.** There is no linear causal chain from stimulus to response. Behavior and perception are locked in a closed loop.

4. **Hierarchical organization.** Complex behavior emerges from many levels of simple control loops, each controlling its own perception by setting goals for the level below.

5. **Reorganization, not reinforcement.** Learning is not the strengthening of specific responses but the random restructuring of control system parameters, driven by intrinsic error, analogous to Ashby's ultrastability and E. coli chemotaxis.

6. **The Test for the Controlled Variable** provides a principled empirical methodology for discovering what an organism is actually doing -- not by observing outputs, but by identifying which perceptual variable is held invariant against disturbance.

7. **Practical parsimony.** In robotics, PCT architectures achieve comparable or superior performance to classical controllers with far simpler designs, no world models, and no inverse kinematics.
