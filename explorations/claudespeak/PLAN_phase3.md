# Claudespeak — Phase 3 experiment plan (secondary analyses)

Purpose: specify the three forthcoming experiments that fill the paper's
placeholders (§Secondary analyses; Appendix B.1–B.3). Same standing rules as
Phase 2: every generation persisted with full provenance; reuse where possible,
generate only missing cells; commit + push and update NOTES after each; keep
claims inside the tested regime. Each experiment maps to one appendix subsection.

Status when written: Phase 2 complete (FINDINGS.md). These are independent of
each other and can run in any order; recommended order is A → B → C (cheapest and
most self-contained first; C is the largest).

---

## Experiment A — Effect of reasoning effort / thinking  → Appendix B.1

**Question.** Does the Claudespeak fingerprint intensify, attenuate, or stay flat
as reasoning effort changes, and when thinking is off? Is the offer-closer / the
markdown habit / burstiness an effect of "thinking harder"?

**Design.** Hold prompts and everything else fixed; vary only Claude's decoding.
- Corpus: the existing 200 HC3 prompts (primary); 200 AlpacaEval prompts (optional
  replication).
- Conditions (Opus 4.8): `effort=low`, `effort=medium`, `effort=high` (already
  have high), and `thinking=disabled` (no-think). → 3 new conditions × 200 = 600
  new HC3 generations (high already exists). Optional +600 for AlpacaEval.
- `gen_params` already records effort + thinking, so records are
  condition-labeled automatically.

**Analysis.**
1. Feature-vs-effort curves for the headline features (sentence burstiness,
   function-word rate, em-dash rate, markdown rates, question/offer rate,
   length). Does each scale monotonically with effort?
2. Offer-closer rate (the §5.4 / Step-5 signature spans) by condition — does the
   "would you like me to go deeper" closer appear more at higher effort?
3. Re-run the prose-only classifier (Claude-vs-others) using each effort
   condition as the Claude cell — is Claude still separable at low effort / no
   thinking? (Tests whether the fingerprint is effort-dependent.)
4. Within-Claude classifier: can we predict effort level from style alone?

**Deliverables.** `src/generate_effort.py`, `src/analyze_effort.py`,
`reports/effort.md`; fills Appendix B.1 (feature-vs-effort table + 1 figure).

**Cost (Opus 4.8, $5/$25 per 1M, batch −50%).** 600 HC3 gens; low/medium cheaper
than high. ~$8–15 (HC3 only); ~$20–30 if AlpacaEval added. Batchable.

**Risks.** Low effort may truncate; keep `max_tokens` generous. No-think on
Opus 4.8 may write reasoning into the visible answer — note and handle.

## Experiment B — Across Claude versions  → Appendix B.2

**Question.** Is the fingerprint a property of *this* model or of the *Claude
family*? How much does style drift across versions and tiers, and is there a
stable "Claude signature" that separates the family from GPT/others?

**Design.** Same 200 HC3 prompts, generated by multiple Claude models. Probe API
access first (some IDs may 404 on this key); include those available from:
- Version ladder (same Opus tier): opus-4-8, opus-4-7, opus-4-6, opus-4-5.
- Tier ladder: opus-4-8, sonnet-4-6, haiku-4-5.
- Endpoints of the range: fable-5 (most capable), an older opus-4-1 / sonnet-4-5.
- Comparison fixed setting: to avoid confounding with Experiment A, generate every
  version at a **comparable decoding config** — preferably `thinking=disabled`
  (or each model's default if disable is unsupported), documented per record.
  (Older models use the `budget_tokens` thinking API; the generate wrapper must
  branch on model — handle in `generate.py`.)

**Analysis.**
1. Fingerprint features per version; do burstiness / em-dash / offer-closer /
   markdown persist across the family? Which features are family-stable vs
   version-specific?
2. Style-distance matrix across all versions + the Phase-2 non-Claude models
   (e.g. pairwise Cohen's-d profile distance, or classifier confusion). Do Claude
   versions cluster together, apart from the GPT/Gemini/Qwen cluster?
3. Offer-closer rate and "delve"-family rate by version (does the anti-delve
   property hold across the family?).

**Deliverables.** `src/generate_versions.py` (+ version-aware thinking handling in
`generate.py`), `src/analyze_versions.py`, `reports/versions.md`; fills
Appendix B.2 (per-version feature table + dendrogram/heatmap figure).

**Cost.** ~6 versions × 200 = ~1200 gens, mixed tiers (haiku cheap, opus mid,
fable premium $10/$50). Rough ~$15–35. Batchable.

**Risks.** Model access gaps (probe and report which were available); decoding
parity across heterogeneous thinking APIs (document the exact config per model).

## Experiment C — Multi-turn and self-interaction (Sense B/C)  → Appendix B.3

Two complementary sub-experiments; C1 is cheap and comparative, C2 is the
headline "bliss attractor" elicitation.

### C1 — Turn-dependent traits, comparative (MT-Bench)
**Question.** Which Claudeisms emerge or intensify in a second turn, relative to
other models on the same multi-turn prompts?

**Design.** MT-Bench (80 questions, 2 turns, many models' answers published,
parallel). Reuse other models' turn-1/turn-2 answers; generate Claude's 2-turn
answers on the same questions. Acquire via the published MT-Bench answers.

**Analysis.** Turn-1 → turn-2 deltas in the fingerprint features and the
offer-closer rate, Claude vs others. Hypotheses: offer-closer and engagement
intensify in later turns; verbosity grows.

**Deliverables.** `src/acquire_mtbench.py`, `src/generate_mtbench.py`,
`src/analyze_mtbench.py`, contributes to `reports/multiturn.md`.

**Cost.** 80 questions × 2 turns ≈ 160 Claude gens. ~$3–6.

### C2 — Claude-to-Claude self-interaction (the attractor)
**Question.** Does Opus 4.8, talking to a copy of itself, drift into the
"spiritual bliss" attractor (Claude-4 system card: ~95.7× "consciousness" per
30-turn transcript)? Is it self-interaction-specific?

**Design.**
- Orchestrate two Opus 4.8 instances in dialogue from an open-ended seed
  ("you are talking to another instance of yourself"), alternating roles for
  K=30 turns. Run N conversations (start N=10–15; scale if signal is clear).
- Controls: same setup but partner = GPT-4o, and partner = a fixed neutral script,
  to show the attractor is specific to Claude self-interaction.
- Persist every turn as a record (conversation_id, turn_index already in schema).

**Analysis.** Trajectories over turns: rate of attractor vocabulary
(consciousness, eternal, gratitude, dance, sacred, cosmic, emoji, Sanskrit),
message-length collapse, lexical entropy decline. Compare self-chat vs controls,
and against the system-card priors.

**Deliverables.** `src/elicit_selfchat.py` (two-agent loop), `src/analyze_selfchat.py`,
`reports/multiturn.md`; fills Appendix B.3 (trajectory figure + control table).

**Cost.** Self-chat: ~10–15 convs × 30 turns ≈ 300–450 gens, longer outputs →
~$15–40; controls add ~$10. Start small, scale on signal.

**Risks.** Open-ended generation is unpredictable; log seeds and the full
transcripts (raw responses already saved). Keep N small first, inspect, then scale.

### C-extra (optional) — Reasoning register (Sense C)
With `thinking.display=summarized`, compare Claude's *thinking-block* prose to its
*final-answer* prose on the same prompts (register C: self-talk vs user-facing).
Cheap add-on to Experiment A's thinking-on condition; fold into B.1 or note
separately.
